Переход на версию ТЯ 7.3.7.5. Проверка работоспособности технологического ядра этой версии требует дополнительных действий.

09/10/2014 v1.1.3

При изготовлении клона гигантской базы пришлось удалять клиентов. Как это сделать понятно из файла insroot.sql. Попутно исправлено несколько ошибок.

14/10/2013 v1.1.2

Адаптация для Oracle 12C. При таком портировании некоторые package из технологического ядра переходят в состояние INVALID с ошибкой PLS-00452. Это не помешает изготовлению "КОПИИ", но работать она не будет. Нужные исправления лучше сделать на стадии Согласование данных.

05/06/2013 v1.1.1

  • Изготовлением "test case" начинают заниматься не от хорошей жизни. Поэтому операция "дополнение данных" была изменена с целью сокращения времени выполнения.
  • Изменен алгоритм восстановления целостности "КОПИИ".
  • Выполняется подготовка "КОПИИ" для работы под логином отличным от владельца (IBS). Достаточно просто перерегистрировать пользователя.

28/03/2013 v1.1.0

Окончательно отвергнута идея восстанавливать целостность "КОПИИ" по алгоритму "ON DELETE CASCADE". Теперь в большинстве случаев будет использоваться алгоритм "ON DELETE SET NULL". Два исключения из этого правила. Первое - внешние ключи с ограничением "NOT NULL". Второе - ID объектов в дочерних классах.

20/11/2012 v1.0.0

Добавлена возможность дополнять "КОПИЮ" данными из "ОРИГИНАЛА".

13/09/2012 v0.9

Впервые выложил в сеть. Вроде стабильная. (Сегодня юбилей моего первого запуска IBSO).

Предыстория

“Жаль...” - выдохнул Андрей. Он отвинтил от лацкана значёк и положил передо мной на ресторанный столик: “На вот... на память”. Через два месяца “ОАО Уралпромстройбанк” прекращал свое существование...

Значёк я сохранил. Вовсе не потому, что он принадлежал человеку ставшему впоследствии одним из руководителей ЦФТ. И даже не потому, что, говорят, он из золота. Я сохранил его в память о времени, когда я был частичкой умного и веселого коллектива с мудрым начальником во главе.

В то время в мои служебные обязанностями входила забота об Oracle вобще и схеме IBSO в частности. Финансированием нас не баловали. (Зарплатой, кстати, тоже.) Потому для разработок и экспериментов изготовлялся уменьшеный функциональный эквивалент системы.
